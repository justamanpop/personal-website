Read List

This is a list of Technical books that have imapcted me greatly and I recommend to every programmer

  • The Pragmatic Programmer - Andy Hunt and Dave Thomas - This is a general book covering many aspects of programming. It taught me a lot of very important fundementals, as well as helped articulate and express concepts I picked up with experience but was not able to concretely explain
  • Unit Testing Principles, Practices and Patterns - Vladimir Khorikov - This is one of the most important books I've read. It taught me in depth about unit testing, changed how I think about and approach writing tests, what kinds of tests to write and how many of each, and much more. It really drove home how cruical tests are and how as much care and effort should be put into maintaining them as with production code
  • Test-Driven Development By Example - Kent Beck - Introduced TDD to me, and now I almost always develop using it. I can't imagine writing code any other way now. TDD is a lot more than just a fad or preference, it really helped me write better code more confidently
  • Refactoring: Improving the Design of Existing Code - Martin Fowler - Taught me the basics of refactoring and how to go about it in a safe way. It taught me the concept of incremental refactoring to improve a codebase just ever so slightly and safely, and reinforced how important tests are
In progress:
